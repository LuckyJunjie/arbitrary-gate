# 时光笺 · 技术架构设计文档

## 一、技术选型：H5 + 云服务 = 完整产品

**核心判断**：这款产品的前端交互高度定制化（卷轴、墨池、手势），而后端逻辑相对清晰（状态管理、AI调用、卡池算法）。H5版本可以完整覆盖所有核心功能，且天然具备以下优势：

| 方案 | 优势 | 劣势 |
|---|---|---|
| **H5（推荐）** | 天然跨平台、分享方便、用户无需下载、迭代快 | 性能受限、功能调用受限 |
| **小程序** | 性能好、微信生态入口 | 需要审核、平台绑定 |
| **App** | 性能最强 | 开发成本高、用户门槛高 |

**结论**：H5版本先行，小程序/App作为后续扩展。所有设计完全支持H5实现，UI动画用CSS/Canvas，性能完全足够。

### 为什么选 H5

1. **分享即入口**：故事卡分享到朋友圈/微信，用户点开直接就是H5，无需下载App，降低了99%的流失率
2. **迭代成本低**：前端代码更新用户无感知，后端API升级后可灰度发布
3. **跨平台**：同一套代码，同时支持iOS Safari、Android浏览器、微信内嵌WebView、PC浏览器
4. **完整能力**：H5 + 云开发（CloudBase/小程序云开发）可以实现所有核心功能：
   - 用户登录（微信授权、手机号一键登录）
   - 支付（微信H5支付）
   - 数据库（云数据库）
   - 存储（云存储）
   - AI能力（云调用AI服务）

## 三、H5 前端技术栈

### 3.1 核心框架

```
Vue3 (Composition API) + TypeScript
├── 状态管理：Pinia（轻量、TypeScript友好）
├── 路由：Vue Router
├── HTTP：Axios / ky
├── 动画：GSAP（复杂动画）+ CSS Animation（简单过渡）
├── 手势：VueUse（useSwipe, usePointerLock）
├── 分享：微信 JSSDK（分享到朋友圈/朋友）
├── 支付：Wechat JSAPI
└── 构建：Vite
```

### 3.2 关键页面与路由

```
/                     # 首页（书房主界面）
/pool                # 墨池抽卡
/cards               # 卡匣（我的卡牌）
/story/:id          # 故事阅读（卷轴）
/story/:id/preview  # 组合预览
/bookshelf          # 书架
/bookshelf/timeline # 时光轴视图
/bookshelf/map      # 山河图视图
/story/:id/share    # 故事卡分享页
/settings           # 设置
```

### 3.3 核心交互实现

**【墨池抽卡动画】**
```
1. 墨面涟漪：CSS animation + SVG circle + filter blur
2. 卡片浮出：CSS transform + clip-path（模拟墨汁滑落效果）
3. 残片拼接：Canvas 涂抹遮罩（destination-out 混合模式）
4. 墨香渐淡：CSS animation + filter: blur(px) 随时间衰减
```

**【卷轴阅读】**
```
1. 竖向排版：writing-mode: vertical-rl（CSS原生支持）
2. 手势滑动：@vueuse/core useSwipe + 惯性滚动
3. 涟漪动画：Canvas 2D + requestAnimationFrame
4. 逐字渲染：WebSocket 接收 + v-html + CSS animation
```

**【手势选择】**
```
1. 手势识别：@vueuse/core usePointerEvents
2. 方向判断：计算起点终点向量夹角
3. 慢速检测：记录滑动轨迹，用时超过阈值才有效
4. 触感反馈：navigator.vibrate([10])（手机震动API）
```

## 四、后端技术栈（云开发版）

### 4.1 推荐：腾讯云开发 CloudBase

```
CloudBase Framework（一套前端框架部署后端）
├── 云函数：Node.js（处理AI调用、卡池逻辑）
├── 云数据库：MongoDB（JSON友好，story表天然适合）
├── 云存储：存储卡图、故事卡图片
├── 登录：微信一键授权
└── 静态托管：Vite构建产物直接部署

优势：免服务器、免运维、按调用量付费，小团队神器
```

### 4.2 备选：自建 Spring Boot（Java团队）

```
Spring Boot 3.x + MyBatis-Plus
├── MySQL 8.0：主数据存储
├── Redis：会话缓存、卡池状态、抽卡冷却
├── Spring AI：统一对接通义/文心/GPT
└── Sa-Token：轻量用户认证
```

### 4.3 AI 服务集成

**文本生成（必须）：**
- 优先：通义千问（阿里云百炼，性价比高）
- 备选：文心一言 / GPT-4o-mini

**图片生成（可选）：**
- 通义万相 / Midjourney API / Stable Diffusion

**Prompt管理：**
- 所有AI Prompt模板存储在数据库/配置中心，支持热更新
- 用户测试时无需重新发版

## 五、数据库设计（简化版）

### 5.1 核心表

```sql
-- 用户表
CREATE TABLE user (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  open_id VARCHAR(64) UNIQUE,  -- 微信openId
  nickname VARCHAR(64),
  avatar_url VARCHAR(512),
  ink_stone INT DEFAULT 100,     -- 墨晶（抽卡消耗）
  created_at DATETIME DEFAULT NOW()

-- 用户卡牌收集
CREATE TABLE user_card (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT,
  card_type TINYINT,            -- 1关键词 2事件
  card_id BIGINT,               -- 关联卡牌库ID
  acquired_at DATETIME DEFAULT NOW(),
  ink_fragrance TINYINT DEFAULT 7,  -- 墨香值(0-7)
  resonance_count INT DEFAULT 0,  -- 累计共鸣次数
  UNIQUE KEY uk_user_card (user_id, card_type, card_id)
);

-- 故事主表
CREATE TABLE story (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  story_no VARCHAR(32) UNIQUE, -- 分享编号
  user_id BIGINT,
  
  -- 入局配置
  event_card_id BIGINT,
  keyword_card_ids JSON,        -- [101, 205, 409]
  identity_type TINYINT,        -- 1高位 2低位 3旁观者
  entry_answers JSON,            -- {"q1":"A","q2":"B","q3":"C"}
  
  -- 元数据
  title VARCHAR(128),
  style TINYINT,               -- 1白描 2江湖 3笔记 4话本
  total_words INT DEFAULT 0,
  history_deviation INT DEFAULT 50,  -- 0-100
  
  status TINYINT DEFAULT 1,       -- 1进行中 2已完成
  current_chapter INT DEFAULT 1,
  created_at DATETIME DEFAULT NOW(),
  finished_at DATETIME
);

-- 故事章节
CREATE TABLE story_chapter (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  story_id BIGINT,
  chapter_no INT,
  scene_text TEXT,              -- 场景描写
  options JSON,                 -- [{"id":1,"text":"退回马厩"}]
  selected_option TINYINT,       -- 用户选择的选项ID
  keyword_resonance JSON,       -- {"cardId1":2, "cardId2":1}
  ripples JSON,                  -- [{"target":"铜锁芯","status":"变凉"}]
  created_at DATETIME DEFAULT NOW(),
  UNIQUE KEY uk_story_chapter (story_id, chapter_no)
);

-- 故事配角
CREATE TABLE story_character (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  story_id BIGINT,
  name VARCHAR(32),
  character_type TINYINT,      -- 1命运羁绊 2历史节点 3市井过客
  fate_value INT DEFAULT 50,    -- 命运值(0-100)

  relation_to_user VARCHAR(64)

-- 手稿内容
CREATE TABLE story_manuscript (
  story_id BIGINT PRIMARY KEY,
  full_text LONGTEXT,          -- 完整小说正文
  annotations JSON,             -- 朱批位置及内容
  choice_marks JSON,            -- 用户选择标记
  epilogue TEXT,               -- 后日谈
  word_count INT DEFAULT 0
);

-- 分享记录
CREATE TABLE story_share (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  story_id BIGINT,
  share_code VARCHAR(32) UNIQUE,
  missing_corner_card_id BIGINT,  -- 缺角对应的关键词卡
  joint_count INT DEFAULT 0,
  created_at DATETIME DEFAULT NOW()

-- 关键词卡牌库（静态数据）
CREATE TABLE keyword_card (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  card_no VARCHAR(32) UNIQUE,
  name VARCHAR(16),             -- 关键词（2-3字）
  category TINYINT,             -- 1器物 2职人 3风物 4情绪 5称谓
  rarity TINYINT,               -- 1凡 2珍 3奇 4绝

  weight INT DEFAULT 100

-- 历史事件卡牌库（静态数据）
CREATE TABLE event_card (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  card_no VARCHAR(32) UNIQUE,
  title VARCHAR(64),           -- "赤壁·东风骤起"
  dynasty VARCHAR(32),
  location VARCHAR(64),
  description TEXT,
  image_url VARCHAR(512)
);
```

## 七、AI Agent协作（H5版）

### 7.1 Agent职责与Prompt

**说书人（场景生成）：**
```
你是一位温和的中年说书人，擅用白描语言。
当前：第{chapter}章，事件：{eventName}，身份：{identity}
关键词：{keywords}，配角：{characters}
上文选择：{lastChoice}

请生成：场景描写(200字) + 情境困境 + 三个选项(JSON)
禁止"宛如""仿佛""无法言说"等词汇。
```

**判官（选项评估）：**
```
当前选择：{optionText}，选项对应价值观：{valueOrient}
配角命运：{characters}，关键词共鸣：{resonance}

请评估：该选择对配角命运的影响，历史偏离度变化（0-100），并给出下一场景描写。
```

**稗官（后日谈生成）：**
```
故事正文：{fullText}，配角：{characters}，历史偏离度：{deviation}

请以"稗官曰"口吻，写一段后日谈（200字），点评与真实历史的异同。
```

## 八、H5特别注意事项

### 8.1 微信环境适配

| 问题 | 解决方案 |
|---|---|
| 微信内嵌浏览器内核 | 使用Vue3 + Vite，配置polyfill |
| 微信分享 | 接入微信JSSDK，配置分享标题+图片+链接 |
| 微信支付 | 接入JSAPI支付（购买墨晶/时光笺道具） |
| 缓存问题 | 微信WebView缓存策略：no-cache + v='{version}' |

### 8.2 性能优化

- **图片懒加载**：使用IntersectionObserver，卡片图仅在可见时加载
- **动画性能**：优先使用CSS transform/opacity，避免重排重绘
- **AI流式渲染**：使用v-html + CSS animation，避免Vue响应式开销
- **骨架屏**：首屏加载时显示"宣纸纹理骨架"，提升感知速度

### 8.3 离线支持

- **Service Worker**：缓存静态资源和已读过的故事章节
- **故事草稿**：localStorage存储进行中故事，本地优先展示

## 九、总结

**H5版本可以实现全部核心功能**，且在以下方面具有原生优势：

- **分享传播**：点开即玩，零下载门槛
- **迭代速度**：前后端均可热更新，无需发版
- **成本**：云开发按量付费，初期成本极低
- **用户体验**：现代H5 + Canvas完全可以实现所有设计动效

**推荐路径**：H5验证产品 → 微信小程序补充（可选）→ 原生App扩展（可选）
