C-14 AI画师后端已**完整实现**，无需额外开发：

#### 现有实现

**1. `ImageService.java`** (`src/backend/com/timespace/module/image/service/ImageService.java`)
- 调用通义万相 `wanx2.1` API (`https://dashscope.aliyuncs.com/api/v1/services/aigc/text2image/image-synthesis`)
- API Key 配置：`spring.ai.dashscope.api-key` (env: `DASHSCOPE_API_KEY`)
- Redis 缓存：相同 prompt SHA-256 hash 24h 不重复生成
- 降级方案：无 API Key 时返回 SVG 占位图 (data URI)
- 返回结构：`ImageGenerateResult { imageUrl, cached }`

**2. `ImageController.java`** (`src/backend/com/timespace/module/image/controller/ImageController.java`)
- 端点：`POST /api/image/generate`
- 请求体：`ImageGenerateRequest { prompt: String, size: String(default "512*768") }`
- 登录要求：`StpUtil.checkLogin()` 但允许游客降级调用
- 返回：`Result<ImageGenerateResult>`

**3. 前端集成** (`src/frontend/services/aiPainter.ts`)
- `AIPainterService.callTongyiWanxiang()` 优先调用 `/api/image/generate` 后端代理
- 降级：前端直连 WanX API（需要 `VITE_TONGYI_API_KEY`）
- 前端缓存：24h 内存缓存 (Map)

#### 与任务要求的差异

任务描述 API 设计使用 `@RequestParam` (query string)，实际实现使用 `@RequestBody` (JSON body)，功能等价，无问题。

任务描述提及 `type` 参数（关键词卡/场景），但当前实现未使用此字段 — 两种图片类型由前端 prompt builder 区分，后端只负责渲染。

  api-key: ${DASHSCOPE_API_KEY:your-api-key}

  options.model: wanx-v1
